Your First Publisher

Introduction​

Time to write code! In Lesson 2, you explored ROS 2 from the command line. Now you'll write a Python node that publishes messages.

Your goal: Write a program that sends movement commands to Turtlesim automatically, making the turtle move without manual commands.

What you'll build:

class TurtleCommandPublisher(Node):
    # Publishes motion commands
    # Timer fires every 0.5 seconds
    # Turtle moves forward, then stops

Understanding Publishers: The Theory​

Before you code, let's review what a publisher does.

Publisher Role​

A publisher is a node that sends messages to a topic.

Your Code (Publisher)
    ↓ sends message "move forward"
    ↓
Topic: /turtle1/cmd_vel
    ↓ subscribers listen here
    ↓
Turtlesim (Subscriber) receives message → moves turtle

Key Elements of a Publisher Node​

  1. Import rclpy — Python library for ROS 2
  2. Create a Node subclass — Your code inherits from rclpy.node.Node
  3. Create a publisher — self.create_publisher(MessageType, '/topic/name', queue_size)
  4. Create a timer — For periodic execution (not required, but common)
  5. Implement timer callback — Function that publishes messages
  6. Initialize rclpy — rclpy.init() before creating nodes
  7. Spin the node — rclpy.spin(node) to keep it running

Layer 2: Creating Your First Publisher Node​

Step 1: Create a File​

Create a new file called publisher_node.py:

touch ~/publisher_node.py

Open it in your text editor (VS Code, nano, vim, etc.).

Step 2: Write the Publisher Code​

Type this code carefully (don't copy-paste—typing builds muscle memory):

import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist


class TurtleCommandPublisher(Node):
    def __init__(self):
        super().__init__('turtle_command_publisher')

        # Create a publisher that sends Twist messages to /turtle1/cmd_vel
        # queue_size=10 means buffer up to 10 messages if subscribers are slow
        self.publisher_ = self.create_publisher(Twist, '/turtle1/cmd_vel', 10)

        # Create a timer that calls timer_callback every 0.5 seconds
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)

        # Counter to track how many times we've published
        self.counter = 0

    def timer_callback(self):
        """Called every 0.5 seconds by the timer"""

        # Create a Twist message (velocity command for the turtle)
        msg = Twist()
        msg.linear.x = 1.0   # Move forward at 1 m/s
        msg.angular.z = 0.0  # Don't rotate

        # Publish the message
        self.publisher_.publish(msg)

        # Log what we published (for debugging)
        self.get_logger().info(f'Publishing: linear.x={msg.linear.x}')

        # Increment counter
        self.counter += 1

        # Stop after 5 publishes (2.5 seconds total)
        if self.counter >= 5:
            # Send a stop command
            stop_msg = Twist()
            stop_msg.linear.x = 0.0
            stop_msg.angular.z = 0.0
            self.publisher_.publish(stop_msg)

            # Log and shutdown
            self.get_logger().info('Turtle stopped. Shutting down.')
            rclpy.shutdown()


def main(args=None):
    """Entry point for the ROS 2 node"""

    # Initialize ROS 2 (must be called before creating nodes)
    rclpy.init(args=args)

    # Create an instance of our publisher node
    node = TurtleCommandPublisher()

    # Run the node (this blocks until rclpy.shutdown() is called)
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        # Clean up
        node.destroy_node()
        rclpy.shutdown()


if __name__ == '__main__':
    main()

Step 3: Understand the Code​

Let me break down each section:

Imports:

import rclpy                           # ROS 2 Python library
from rclpy.node import Node            # Base class for nodes
from geometry_msgs.msg import Twist    # Message type for velocity commands

Class Definition:

class TurtleCommandPublisher(Node):    # Inherit from Node
    def __init__(self):                # Constructor
        super().__init__('turtle_command_publisher')  # Set node name

Publisher Creation:

self.publisher_ = self.create_publisher(
    Twist,                # Message type
    '/turtle1/cmd_vel',   # Topic name
    10                    # Queue size
)

Timer Creation:

self.timer = self.create_timer(
    0.5,                      # Call every 0.5 seconds
    self.timer_callback       # Function to call
)

Callback Function:

def timer_callback(self):
    # Called every 0.5 seconds
    msg = Twist()
    msg.linear.x = 1.0      # Set forward velocity
    self.publisher_.publish(msg)  # Send it

Main Function:

def main(args=None):
    rclpy.init(args=args)     # Initialize ROS 2
    node = TurtleCommandPublisher()  # Create node
    rclpy.spin(node)          # Run the node

Step 4: Run the Publisher​

Make sure:

  • Terminal 1: Turtlesim running (ros2 run turtlesim turtlesim_node)
  • Terminal 2: Your publisher node

In Terminal 2, run:

cd ~
python3 publisher_node.py

Expected output:

[INFO] [turtle_command_publisher]: Publishing: linear.x=1.0
[INFO] [turtle_command_publisher]: Publishing: linear.x=1.0
[INFO] [turtle_command_publisher]: Publishing: linear.x=1.0
[INFO] [turtle_command_publisher]: Publishing: linear.x=1.0
[INFO] [turtle_command_publisher]: Publishing: linear.x=1.0
[INFO] [turtle_command_publisher]: Turtle stopped. Shutting down.

In Terminal 1 (Turtlesim window):

  • Turtle should move forward for ~2.5 seconds, then stop
  • You should see a pen trail (default: turtle draws as it moves)

💬 AI CoLearning: Understanding Message Types​

Your code used Twist message type. This is important!

Ask Claude:

  1. "What's inside a Twist message? What fields does it have?"
  2. "Why do we need to set msg.linear.x instead of just msg = Twist(1.0, 0.0)?"
  3. "What would happen if I set msg.angular.z = 1.0 instead?"

Expected answers:

  1. Twist has linear (x, y, z velocity) and angular (x, y, z rotation)
  2. Twist is a structured message with named fields; we set them by name
  3. The turtle would rotate instead of move forward

This teaches you about message structures—the key to ROS 2 communication.

Experiment: Modify the Publisher​

Now that you have a working publisher, modify it:

Experiment 1: Make the Turtle Faster​

Change this line:

msg.linear.x = 1.0   # Change to 2.0

To:

msg.linear.x = 2.0   # Move faster

Run again. The turtle should move faster.

Experiment 2: Make the Turtle Rotate​

Change:

msg.linear.x = 1.0
msg.angular.z = 0.0

To:

msg.linear.x = 0.0      # Don't move forward
msg.angular.z = 1.0     # Rotate at 1 rad/s

Run again. The turtle should spin in place.

Experiment 3: Move in a Square​

Modify timer_callback to alternate between moving and rotating:

def timer_callback(self):
    msg = Twist()

    # Alternate: move forward, then rotate
    if self.counter % 2 == 0:
        msg.linear.x = 1.0   # Move forward
        msg.angular.z = 0.0
    else:
        msg.linear.x = 0.0
        msg.angular.z = 0.5  # Rotate

    self.publisher_.publish(msg)
    self.counter += 1

    if self.counter >= 20:
        # Stop
        stop_msg = Twist()
        self.publisher_.publish(stop_msg)
        rclpy.shutdown()

Run again. The turtle should move forward, rotate, move forward, rotate, etc.

Verifying Your Publisher with ros2 topic echo​

While your publisher is running, open a third terminal and watch the topic:

ros2 topic echo /turtle1/cmd_vel

Expected output:

linear:
  x: 1.0
  y: 0.0
  z: 0.0
angular:
  x: 0.0
  y: 0.0
  z: 0.0
---
linear:
  x: 1.0
  y: 0.0
  z: 0.0
angular:
  x: 0.0
  y: 0.0
  z: 0.0
---

Each --- separator shows a new message. You should see one message every 0.5 seconds (matching your timer).

This proves:

  • Your publisher is sending messages ✅
  • The messages are correct (linear.x = 1.0) ✅
  • Turtlesim is receiving them (turtle moves) ✅

Troubleshooting​

Error: "ModuleNotFoundError: No module named 'geometry_msgs'"​

Cause: geometry_msgs not installed

Solution:

sudo apt install ros-humble-geometry-msgs

Error: "Cannot find node named '/turtlesim'"​

Cause: Turtlesim not running

Solution:

  • Open Terminal 1 and run: ros2 run turtlesim turtlesim_node
  • Then run your publisher in Terminal 2

Error: "rclpy.init() called twice"​

Cause: You ran the publisher multiple times in the same Python process

Solution:

  • This shouldn't happen unless you're importing the module. Just run the script cleanly: python3 publisher_node.py

Error: "Turtle doesn't move"​

Diagnosis steps:

  1. Is Turtlesim running?

    ros2 node list  # Should show /turtlesim

  2. Is your publisher running?

    ros2 node list  # Should show /turtle_command_publisher

  3. Are messages being published?

    ros2 topic echo /turtle1/cmd_vel  # Should show Twist messages

  4. Is the topic name correct?

    ros2 topic list  # Should show /turtle1/cmd_vel

If all checks pass and turtle still doesn't move, the issue might be a message value. Try running the manual command from Lesson 2:

ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "linear: {x: 1.0}"

If the turtle moves with this command but not with your publisher, the message structure might be wrong. Compare your Twist() creation to the manual command above.

Turtle Moves But Doesn't Stop​

Cause: Your code might not have reached the if self.counter >= 5 stop condition

Solution: Press Ctrl+C to stop the node, or add a print statement to debug:

self.get_logger().info(f'Counter: {self.counter}')  # Add this line

🤝 Collaboration Exercise: Debug This!​

Here's intentionally broken code. Use the diagnostic tools above to find the problem and fix it:

# ❌ BROKEN CODE (intentionally)
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import Twist

class BrokenPublisher(Node):
    def __init__(self):
        super().__init__('broken_publisher')
        self.publisher_ = self.create_publisher(Twist, '/turtle1/cmd_vel', 10)
        # MISSING: self.timer = self.create_timer(...)
        self.counter = 0

    def timer_callback(self):
        msg = Twist()
        msg.linear.x = 1.0
        self.publisher_.publish(msg)
        self.counter += 1

def main(args=None):
    rclpy.init(args=args)
    node = BrokenPublisher()
    rclpy.spin(node)

if __name__ == '__main__':
    main()

Your task:

  1. Run this code (even though it's broken)
  2. Use ros2 topic echo /turtle1/cmd_vel to check if messages are published
  3. Use ros2 node list to verify the node exists
  4. Figure out what's wrong
  5. Ask Claude for a hint if stuck
  6. Fix the code
  7. Verify it works

What's wrong? The timer is never created, so timer_callback is never called. Fix: Add self.timer = self.create_timer(0.5, self.timer_callback) to __init__.
