Setting Up Your Environment

Introductionâ€‹

Now that you understand what ROS 2 is, let's get it running on your computer.

This lesson is all about setup: installing ROS 2, launching Turtlesim (your first robot simulator), and exploring the system with command-line tools. By the end, you'll see nodes and topics working in a real ROS 2 system.

Time estimate: 30â€“45 minutes (depending on internet speed for downloading packages)

Prerequisitesâ€‹

Before starting:

  • Operating System: Ubuntu 22.04 LTS (this guide assumes Ubuntu; adapt if using WSL2 on Windows)
  • Internet Connection: Required for downloading ROS 2 packages
  • Terminal: Comfortable opening and running commands
  • Administrator Access: sudo privileges (you'll install packages)

Step 1: Install ROS 2 Humbleâ€‹

If ROS 2 is already installed on your computer, skip to "Verify Installation" below.

Installation Steps (Ubuntu 22.04)â€‹

Open a terminal and run the following commands in order:

1. Set locale

locale  # Check for UTF-8

If you don't see UTF-8, run:

sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

2. Add ROS 2 repository

sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg

3. Add ROS 2 source list

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

4. Install ROS 2 Humble

sudo apt update
sudo apt upgrade
sudo apt install ros-humble-desktop

This downloads ~2â€“3 GB of packages. Estimated time: 10â€“20 minutes (depends on internet speed).

5. Set up environment

source /opt/ros/humble/setup.bash

To make this automatic every time you open a terminal:

echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc
source ~/.bashrc

Step 2: Verify Installationâ€‹

Check that ROS 2 is installed correctly:

ros2 --version

Expected output:

ROS 2 humble (12.x.x - the exact version doesn't matter)

If you see an error like ros2: command not found, installation didn't complete. Re-run the installation steps above.

Step 3: Install Turtlesimâ€‹

Turtlesim is a simple 2D robot simulator. It comes with ROS 2, but you need to install the package:

sudo apt install ros-humble-turtlesim

Verify installation:

ros2 run turtlesim turtlesim_node --help

Expected output:

usage: turtlesim_node [-h] [--ros-args ...]
...

If you see help text, Turtlesim is installed correctly.

Step 4: Launch Turtlesimâ€‹

Now let's run your first ROS 2 node!

Terminal 1 (Turtlesim window):

ros2 run turtlesim turtlesim_node

Expected output:

[INFO] [turtlesim]: Starting turtlesim_node with pid [XXXX]

Plus a new window should open showing:

  • A light blue background
  • A small orange turtle in the center
  • Nothing moving yet

If the window opens, Turtlesim is running. Leave it open.

Step 5: Explore with Command-Line Toolsâ€‹

Open a new terminal (Terminal 2). Keep Turtlesim running in Terminal 1.

Command 1: List Running Nodesâ€‹

ros2 node list

Expected output:

/turtlesim

This shows one node is running: turtlesim. Its full name is /turtlesim.

Command 2: List Active Topicsâ€‹

ros2 topic list

Expected output:

/turtle1/cmd_vel
/turtle1/color_sensor
/turtle1/pose
/turtlesim/describe_parameters
/turtlesim/get_parameter_types
/turtlesim/get_parameters
/turtlesim/list_parameters
/turtlesim/set_parameters
/turtlesim/set_parameters_atomically

These are topics the Turtlesim node is publishing and subscribing to. The important ones for us:

  • /turtle1/cmd_vel â€” Where we send movement commands
  • /turtle1/pose â€” Where Turtlesim publishes the turtle's position
  • /turtle1/color_sensor â€” Where Turtlesim publishes the color under the turtle

Command 3: Get Information About a Topicâ€‹

ros2 topic info /turtle1/cmd_vel

Expected output:

Type: geometry_msgs/msg/Twist
Publisher count: 0
Subscriber count: 1

This tells you:

  • Type: The kind of message sent on this topic (Twist is a velocity command)
  • Publishers: 0 nodes are publishing to this topic (yet)
  • Subscribers: 1 node is subscribing (the turtle simulator, waiting for commands)

Command 4: Echo a Topic (Watch Data Flow)â€‹

Let's watch the turtle's position in real-time:

ros2 topic echo /turtle1/pose

Expected output (continuous):

x: 5.544444561004638
y: 5.544444561004638
theta: 0.0
linear_velocity: 0.0
angular_velocity: 0.0
---

The --- separator shows each update. Right now, the turtle isn't moving, so values don't change.

To stop watching: Press Ctrl+C in Terminal 2.

Step 6: Make the Turtle Moveâ€‹

Let's send a movement command. In Terminal 2 (after stopping the echo from above):

ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "linear: {x: 1.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}"

In Terminal 1 (Turtlesim window):

  • The turtle should move forward

What just happened:

  1. You published a message to /turtle1/cmd_vel
  2. The message: linear.x = 1.0 (move forward at 1 m/s)
  3. Turtlesim subscribed to this topic and received the message
  4. The turtle moved

This is the pub/sub pattern in action:

  • You = publisher (sent a message)
  • /turtle1/cmd_vel = topic (the channel)
  • Turtlesim = subscriber (received and acted on it)

Note: The turtle will keep moving until you send a stop command or Ctrl+C the pub command. To stop:

# Press Ctrl+C to stop the pub command
# Then send a stop command:
ros2 topic pub /turtle1/cmd_vel geometry_msgs/msg/Twist "linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}"

ðŸ’¬ AI CoLearning: Understanding What Just Happenedâ€‹

At this point, you've seen nodes, topics, and pub/sub in action. Some questions to deepen understanding:

Ask Claude:

  1. "I just published a command to move the turtle. How did the Turtlesim node know to read that message?"
  2. "Why is the message type geometry_msgs/msg/Twist? What's in a Twist message?"
  3. "Could I have multiple nodes publishing to /turtle1/cmd_vel? What would happen?"

Expected answers from Claude:

  1. Turtlesim subscribed to that topic and gets notified when messages arrive
  2. Twist has linear velocity (x, y, z) and angular velocity (x, y, z)
  3. Yes, but the last message received wins (no guaranteed ordering)

This teaches you to think about the system conceptually, not just memorize commands.

Step 7: Explore Furtherâ€‹

List All Nodes with Detailsâ€‹

ros2 node list --all

Get Info About a Nodeâ€‹

ros2 node info /turtlesim

Expected output:

Node /turtlesim
  Publishers:
    /turtle1/color_sensor: std_msgs/msg/UInt32
    /turtle1/pose: turtlesim/msg/Pose
  Subscribers:
    /turtle1/cmd_vel: geometry_msgs/msg/Twist
  Services:
    /clear: turtlesim/srv/Clear
    ...

This shows exactly what /turtlesim publishes (topics it sends to) and subscribes to (topics it reads from).

Watch the Topic Dataâ€‹

While moving the turtle in Terminal 1, run this in Terminal 2:

ros2 topic echo /turtle1/pose

Expected output:

x: 7.12
y: 5.54
theta: 0.0
linear_velocity: 1.0
angular_velocity: 0.0
---

The x and y values change as the turtle moves. The linear_velocity shows the speed.

Troubleshootingâ€‹

Error: "ros2: command not found"â€‹

Cause: ROS 2 not installed or environment not set up

Solution:

  1. Verify ROS 2 is installed: ls /opt/ros/humble
  2. If not there, re-run installation steps
  3. Source the setup file: source /opt/ros/humble/setup.bash
  4. Add to .bashrc: echo "source /opt/ros/humble/setup.bash" >> ~/.bashrc

Error: "Could not find executable named 'turtlesim_node'"â€‹

Cause: Turtlesim not installed

Solution:

sudo apt install ros-humble-turtlesim

Error: "Turtlesim window opens but nothing happens"â€‹

Cause: Display server issue (common in SSH/WSL without X forwarding)

Solution:

  • If using SSH: Enable X forwarding: ssh -X user@host
  • If using WSL: Install VcXsrv and set export DISPLAY=localhost:0.0
  • If using native Ubuntu: This shouldn't happen; restart terminal

Error: "Multiple Turtlesim windows open"â€‹

Cause: You ran turtlesim_node multiple times in different terminals

Solution: Each node needs a unique name:

ros2 run turtlesim turtlesim_node --ros-args --remap __node:=turtlesim2

But for now, close extra windows and keep only one.

Self-Assessmentâ€‹

Before moving to Lesson 3, verify:

  • ROS 2 installed and ros2 --version works
  • Turtlesim installed and launches
  • Can see Turtlesim window with turtle in center
  • Can run ros2 node list and see /turtlesim
  • Can run ros2 topic list and see multiple topics
  • Can echo /turtle1/pose and see changing values
  • Successfully moved the turtle with ros2 topic pub
  • Understand that the turtle's movement was the publisher (you) â†’ topic (/turtle1/cmd_vel) â†’ subscriber (Turtlesim)

If all checkboxes pass, you're ready for Lesson 3.
